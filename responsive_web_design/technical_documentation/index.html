Introduction

C++ is a general purpose programming language created as an extension of the C language. Originally it was called "C with Classes," but as more features were integrated C++ was chosen as the name. It is most often implemented as a compiled language. It is a high-level language, but does also facilite low-level memory manipulation. It is perhaps most famous today for its possibility for high performance and thus usage in video game development.

This documentation provides basic information about C++, and you should have a general understanding of programming to get the most out of the content.

History

The creator of C++, Bjarne Stroustrup, began working on "C with Classes" in 1979. The language was created because Stroustrup wanted to integrate features from languages he found useful, but not strong enough on their own, especially Simula, which was good for large software development, but too slow, and BCPL, which was fast, but too low-level. Stroustrup chose C as his basis as it was general-purpose, fast, portable and already in wide use. C++ was created in 1982, as an updated version of "C with Classes." The name was chosen as "++" is in C as with many other languages the increment operator: C++ was the next step up from C.

C++ has been on a three-year update schedule ever since 2011, and has had three stable releases after the original version; C++03, C++98 and C++2.0. The 1998-release was the first standardized version, under a specific ISO group.

Further development of the C++ language follows a guiding philosophy, which consits of a set of principles:

  • It must be driven by actual problems, and features should be immediately useful.
  • Every feature should be implementable
  • Programmers should be able to freely pick any programming style, and have that style supported by C++
  • Allowing useful features take priority over preventing misuse of the language
  • It should provide facilities for organizing and creating code in separate and well-defined parts, as well as tools for joing said parts together
  • There should be no implicit violations of the type system
  • User created types must have the same support and performance as built-in types
  • Unused features should not affect performance negatively
  • There should be no language beneath C++, except for platform specific assembly language.
  • C++ should work alongside other programming languages, and not foster its own incompatible environment
  • If the programmer's intent is unknown, the programmer should be able to specify it via manual control

C++, C, C#

C++ is considered a superset of C, as all valid C-code is also valid C++ code. C#, although having a similar name is built on a different principle, and has features like garbage-collection that would violate C and C++'s fundamental principles about user-control. C# is a younger language, and besides syntax shares little with C and C++. C# is also mostly just used on Windows, while C and C++ can run in any environment that offers a suitable compiler.

Hello World

C++ can be quite the complex language to learn, especially if programming is new to you. To demonstrate this we can look at a common way to write the "Hello World!" program:

#include <iostream>

int main() {
    std::cout << "Hello World" << std::endl;
    return 0;
}

Let's start at the top. #include is a pre-processor directive that takes whatever file comes after and simply pastes the file in place. Yes, iostream is a file without an extension. The angle brackets are typically used to tell the editor that the given file is to be found in the pre-processors include path. That would make sense when we're using a standard library header, like iostream. You can also create your own header files and use normal double quotes to include a file from a relative path. This also introduced us to the C++ pre-processor, which is what runs before the compiler.

So, what's up with int main()? Well, all C++ programs require a point of entry, the entry function. Usually this is preset as main. Some editors let you pick your own entry function. The int part signifies the return type of the function, and will be the exit code of our program. The last function to stop executing given proper crash free execution is always the entry function, so setting up various return statements given conditions can help with debugging and understanding how our program executed. 0 is usually picked as the default success code, and thus we also have a return 0; statement before the end of our function. Some compilers will allow you to omit the return statement for cleaner code. This does not mean that it is not there, but the compiler puts it there for you.

Okay so now the line we haven't spoken about. There's a lot more to unpack here. First, let's take a look at std::cout <<. Here we have a function call. Yes, not exactly an intuitive syntax. First we are accessing the namespace called std, using the name and the namespace access operator. That would be the two colons. Inside std we call the << operator on the cout class. Think of operators as functions. C++ supports operator overloading, so this bitshift operator now functions as the insertion operator. The text-string "Hello World!" is actually the function argument. Next comes another insertion operator. Imagine this as chaining functions like you would normally do with a .someFunc(). std::endl is an object also in the std namespace that does some things to the current iostream, like flushing the stream and performing a carriage return. As you might understand, you can take a deep dive into any line here and be stuck for a few hours. This complexity is what turns some people off, but is also what allows a really competent programmer to create lightning fast code. So, here's a statement by statement explanation of the "Hello World!"-program:

#include <iostream> //copies the file 'iostream' into the document.

int main() { //start of definition of our main-function, the point where the program will start execution. 
   std::cout << //call "<<" on cout in the std namespace.
   "Hello World!" << //pass "Hello World!" as the argument, and make another call to "<<"
   std::endl; //pass 'endl' as argument, and terminate expression with semi-colon

   return 0; //return 0 as program exit code
} //end of definition of main-function
